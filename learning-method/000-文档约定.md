# learning-method/ 文档约定

> 本文档是目录命名、文件命名和扩展规则的唯一依据。
> 使用说明（AI 执行入口与协议）见 `README.md`。

---

## 1. 文件命名规则

### 1.1 本目录文件

| 规则               | 说明                            | 示例                      |
| ------------------ | ------------------------------- | ------------------------- |
| `{序号}-{标题}.md` | 序号两位，`01` 起，决定阅读顺序 | `01-仿写学习法-design.md` |

### 1.2 模板文件

| 规则            | 说明                     | 示例              |
| --------------- | ------------------------ | ----------------- |
| `{功能}模板.md` | 功能描述用中文，清晰即可 | `仿写计划模板.md` |

### 1.3 项目产出文档

当你应用仿写学习法学习某个技术栈时，产生的文档按以下格式命名：

| 文档类型 | 格式                      | 示例                                  |
| -------- | ------------------------- | ------------------------------------- |
| 仿写计划 | `{日期}-{序号}-{标题}.md` | `2026-06-28-仿写计划.md`              |
| 阶段总结 | `{日期}-{序号}-{标题}.md` | `2026-06-29-04-阶段四.md`             |
| 差距分析 | `{日期}-{序号}-{标题}.md` | `2026-06-29-05-阶段回顾与差距分析.md` |

各部分说明：

| 部分     | 规则                                             |
| -------- | ------------------------------------------------ |
| `{日期}` | `YYYY-MM-DD`，实际写这篇笔记的日期               |
| `{序号}` | 两位数字，当天第几篇。当天只写一篇时序号可省略   |
| `{标题}` | 简短中文描述，如"阶段四"、"仿写计划"、"差距分析" |

> 带日期的好处：一看文件名就知道哪天学的、当天进度如何。按文件名排序就是时间线。

## 2. 目录结构

```
learning-method/               ← 方法论和模板
├── README.md                  ← 使用入口
├── 000-文档约定.md             ← 本文档
├── 01-仿写学习法-design.md    ← 设计背景
├── 02-仿写学习法.md           ← 方法论正文
└── templates/                 ← 模板，只放模板不填内容

docs/<技术栈>/                 ← 具体学习案例
├── 000-文档约定.md             ← 该案例自己的约定（按需，可选）
├── {日期}-仿写计划.md
├── {日期}-{序号}-阶段一.md
├── {日期}-{序号}-阶段二.md
├── ...
└── {日期}-{序号}-差距分析.md
```

## 3. 扩展规则

- 新增方法论相关文档 → 放 `learning-method/` 根目录，延续序号
- 新增模板文件 → 放 `templates/` 下
- 新增学习案例 → 新建 `docs/<技术栈>/` 目录（裸名，不加数字前缀）
- 按需为某个学习案例新增 `docs/<技术栈>/000-文档约定.md`，仅在该案例需要额外命名、目录或阶段规则时创建
- 项目产出文档 → 一律按日期命名，不要堆在 templates 里
